# 0.1.0

## 变更概述

- 新增 `mock/generate-mock-data.mjs`，用于从 `mock/output/_crawl_index.json` 和对应 Markdown 自动生成分层 TS mock 数据。
- 新增 `mock/types.ts`，统一约束抓取索引、导航、链接、图片、页面与集合类型。
- 按业务层级生成 `mock/meta`、`mock/index`、`mock/about`、`mock/majors`、`mock/teachers`、`mock/research`、`mock/training`、`mock/military`、`mock/partners`、`mock/enrollment`、`mock/alumni`、`mock/articles` 共 50 个 TS mock 文件。
- 新增 `docs/superpowers/specs/2026-04-09-mock-data-structure-design.md` 与 `docs/superpowers/plans/2026-04-09-mock-data-structure.md`，记录本次设计和执行计划。

## 结构策略

- 以 `docs/` 和 `AGENTS.md` 作为工程约束来源。
- 以 `mock/output/` 作为内容唯一来源。
- 顶层栏目按站点业务层级拆分，而不是按原始 URL 镜像。
- 每条生成记录保留 `sourcePath`、`sourceUrl`、`sourceTitle`，确保可追溯回原始 Markdown。
- `info/<code>/...` 详情页同时归档到对应业务栏目文件和 `mock/articles/*.ts` 聚合文件。
